The Car Maintenance & Service Booking System is a full-stack software project designed to simulate a real-world vehicle service management platform. It models how car owners, mechanics, and administrators interact within a structured booking and maintenance workflow, with a strong focus on backend logic and database design.

Car owners can register vehicles and book maintenance or repair appointments. Mechanics are able to view assigned jobs, manage schedules, and update service statuses, while administrators oversee users, services, and system operations. The project prioritizes realistic use cases commonly found in service-based applications.

A key objective of this system is to apply industry-relevant development practices, including role-based access control, relational database constraints, and clear separation of concerns. Emphasis is placed on data integrity, clean code structure, and maintainability to ensure the system can be extended as new features are added.

Key Features

User registration and role management (car owner, mechanic, administrator)

Vehicle registration and management

Appointment booking and scheduling

Service status tracking

Relational database design with enforced constraints

Technologies Used

Java – backend logic and application structure

SQL / SQLite – database design and data management

PHP – server-side interaction

HTML & CSS – frontend structure and styling

Assembly – low-level programming concepts
